## Spacing Scale

Declared values (all multiples of 4px):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding (px-1.5 py-0.5) |
| sm | 8px | Compact element spacing, section padding (px-4) |
| md | 16px | Default element spacing, card padding |
| lg | 24px | Major section padding (pb-2, gap-2) |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Page-level section breaks |
| 3xl | 64px | Full-page vertical spacing |

**Exceptions:**
- **Touch target minimum:** 44px × 44px for interactive elements (drop zone button, file input overlay) — meets WCAG AAA touch accessibility
- **Progress bar height:** 4px during determinate phase (visual feedback), 24px spinner diameter during indeterminate phase (sufficient visual prominence)

**Implementation notes:**
- Use Tailwind spacing scale: `gap-4`, `px-4`, `py-2`, `space-y-4`, etc.
- Card padding: `p-6` (24px) for major cards (success/error summary)
- Drop zone padding: `p-8` (32px) to create spacious, inviting area
- Icon-to-text gaps: `gap-2` (8px) within buttons and labels

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 | 1.5 | Default text, error details, file names |
| Label | 12px | 600 | 1.25 | Field labels, badge text, helper copy |
| Heading (H3) | 18px | 700 | 1.2 | Section headings ("Import successful", "Import failed") |
| Heading (H4) | 14px | 600 | 1.25 | Subsection headings (KPI delta column headers) |
| Value/Display | 24px | 700 | 1 | Large numeric values (KPI deltas: before/after), row counts |

**Tailwind classes mapping:**
- Body: `text-sm` (14px), `font-normal` (400)
- Label: `text-xs` (12px), `font-semibold` (600)
- Heading (H3): `text-lg` (18px), `font-bold` (700)
- Heading (H4): `text-sm` (14px), `font-semibold` (600)
- Value/Display: `text-2xl` (24px), `font-bold` (700)

**Line height specifics:**
- Body text: 1.5 ratio (21px computed) for readability in error tables and descriptions
- Headings: 1.2 ratio for tighter, more impactful look
- Values: 1 ratio (tight) to stack delta columns vertically without extra space

**Font family:** Inter (system-ui fallback) — already configured in tailwind.config.ts, no changes needed

## Color

**Design principles:**
- **60% dominant surface:** White (`bg-background`) in light mode, slate-950 (`bg-slate-950`) in dark mode (Phase 1 CSS variables)
- **30% secondary surfaces:** Card backgrounds (`bg-card`), input backgrounds (`bg-input`), muted areas
- **10% accent:** ACM blue (brand-600 / brand-700) reserved for primary CTAs and critical interactive elements only
- **Semantic colors:** Red (destructive) for errors, green for success, yellow for warnings

### Color Assignments

| Element | Light Mode | Dark Mode | Notes |
|---------|-----------|-----------|-------|
| **Background (60%)** | white | slate-950 | CSS custom property `--background` |
| **Card surfaces (30%)** | white / gray-50 | slate-900 | `--card` for success/error summary cards |
| **Primary button** | ACM blue (brand-600) | ACM blue (brand-700) | `bg-primary`, white text; used only for "Go to Dashboard" CTA |
| **Secondary button** | slate-200 border + slate-900 text | slate-700 border + slate-100 text | Used for "Upload another file" or "Try another file" |
| **Drop zone border** | slate-300 dashed | slate-600 dashed | Visual affordance for drag-drop interaction |
| **Drop zone hover** | brand-50 bg (light blue) | slate-800 bg | Subtle highlight on hover/drag-over |
| **Success card border** | green-200 | green-900 | Semantic success indicator (left border or border-2) |
| **Success check icon** | green-600 | green-500 | Lucide `CheckCircle2` filled |
| **Error card border** | red-200 | red-900 | Semantic error indicator (left border or border-2) |
| **Error icon** | red-600 | red-500 | Lucide `AlertCircle` filled |
| **Progress bar (determinate)** | brand-600 (ACM blue) | brand-500 | Fills left-to-right, matches primary color |
| **Progress spinner (indeterminate)** | brand-600 | brand-500 | Rotating spinner during "Parsing & validating…" phase |
| **KPI delta positive** | green-600 text + green-100 bg (optional) | green-500 text | Up arrow + green for: inventory value ↑, days-on-hand ↑ |
| **KPI delta negative** | red-600 text + red-100 bg (optional) | red-500 text | Down arrow + red for: opposite of positive cases |
| **KPI delta neutral** | slate-500 text | slate-400 text | Dash icon (—) when delta is zero |
| **Text: primary** | slate-900 | slate-50 | Body text, headings, labels |
| **Text: secondary** | slate-600 | slate-400 | Muted text, helper copy, "parsing…" label |
| **Border: default** | slate-200 | slate-700 | Table borders, input focus rings |
| **Link color** | brand-600 (underlined) | brand-400 (underlined) | If any links appear in error details |

**Accessibility notes:**
- Text contrast in light mode: slate-900 on white meets 19:1 (exceeds 4.5:1 minimum)
- Text contrast in dark mode: slate-50 on slate-950 meets 16:1
- Success/error indicators always paired with icons AND text labels — color never sole signal
- Drag-over state uses opacity/background changes, not color alone

**No custom colors needed** — all values use existing shadcn/ui + Tailwind + Phase 1 CSS variables.

## Layout & Responsive

### Desktop (≥1024px)

- **Page container:** `max-w-2xl` centered (same as dashboard for consistency)
- **Drop zone:** full width, `py-12` vertical padding, `px-8` horizontal
- **Cards (success/error):** full width below drop zone, `max-w-2xl`, `p-6` padding
- **KPI delta grid:** 2×2 grid of KPI cards (4 total), `gap-4` between items
- **Error detail table:** scrollable container, `max-height: 400px`, horizontal scroll if needed

### Tablet (768px–1023px)

- Same layout as desktop; all elements reflow to fit viewport width
- No horizontal scroll on error table (reduce max-height to ~300px if needed)
- Spacing: maintain `px-4` / `gap-4` minimum

### Mobile (< 768px)

- **Drop zone:** full width, viewport-relative padding `px-4 py-8`
- **Cards:** full width with `mx-4` margin (not `max-w-` constraint)
- **KPI delta grid:** stack to single column (1×4) instead of 2×2
- **Error table:** horizontal scroll within container, columns compressed:
  - Hide "Value" column on very small screens (< 640px), show inline in "Reason" instead
  - Or use a collapsible row detail pattern (lower priority for v1)
- **Button size:** keep `h-10` (44px touch target minimum) on primary CTAs

## Interaction & States

### Drop Zone Interaction

**States:**
1. **Idle:** Dashed border (slate-300 light / slate-600 dark), hover to highlight
2. **Hover:** Light background (brand-50 light / slate-800 dark), border becomes solid
3. **Drag-over:** Solid border (brand-400), background more prominent
4. **Active (uploading):** Opacity 0.5, "Uploading…" state text visible
5. **Disabled (concurrent upload):** Opacity 0.5, error message shown

**Keyboard & Accessibility:**
- Drop zone is a `<button>` or has `role="button"` + `tabIndex={0}`
- Click/Enter/Space opens native file picker (`<input type="file" hidden>`)
- Drag-drop listeners on the button's parent container
- Focus ring: visible (browser default or shadcn ring style)

### Progress Indicator

**Determinate phase (upload bytes):**
- Use shadcn `Progress` component set to current byte percentage
- Label below: `"Uploading {filename}… {percent}%"`
- Duration: ~1-3 seconds for typical 1-5 MB file (user sees it)

**Indeterminate phase (parsing & validating):**
- Switch from progress bar to spinner (Lucide `Loader2` with `animate-spin`)
- Label: `"Parsing & validating… this usually takes a second"`
- No percentage; indicates "work is happening"

**Transition:** Once upload body finishes (`xhr.upload.onload`), immediately switch states without clearing the UI.

### Success Summary Card

**Visual design:**
- Green left border (4px) or border-2 all around
- Light green background (brand-50 light / green-950/10 dark, optional)
- Green check icon (Lucide `CheckCircle2`)
- Two-column layout: "Before" | "After" | "Change" headers, 4 KPI rows below

**Interaction:**
- "Go to Dashboard" button: navigates to `/` and invalidates React Query cache for `/api/v1/kpi/summary`
- "Upload another file" button: resets drop zone to idle state, clears success card from view

**Announcement:** `aria-live="polite"` on success card for screen readers

### Error Summary Card

**Visual design:**
- Red left border (4px) or border-2 all around
- Light red background (red-50 light / red-950/10 dark, optional)
- Red alert icon (Lucide `AlertCircle`)
- Grouped error summary above table
- Scrollable table below (max-height 400px)

**Interaction:**
- "Copy all errors" button: copies table content as tab-separated text to clipboard (use `navigator.clipboard.writeText()`)
- "Try another file" button: resets drop zone to idle, clears error card from view
- Table rows: no drill-down in v1 (static display only)

**Announcement:** `aria-live="assertive"` on error card for immediate screen reader notification

### Form Validation (Client-Side)

- **File extension:** checked before upload, inline error message above drop zone if violated
- **File size:** checked before upload, inline error message
- **Multiple files:** take first, show warning toast (or inline message)
- **No form submission:** all validation happens in React handlers, no HTML form validation needed

## Accessibility Requirements

Inherited from Phase 1 & Phase 3, plus upload-specific rules:

### Critical (WCAG AA / AAA)

- [ ] **Drop zone touch target:** minimum 44px × 44px (current design uses button-style interaction — verify height/width)
- [ ] **Focus visible:** drop zone and all buttons have visible focus ring (`:focus-visible` or `:focus`)
- [ ] **Color not sole signal:**
  - Success: green color + green check icon + "successful" text label
  - Error: red color + red alert icon + "failed" text label
  - Delta: green text + up arrow + "+" sign for positive; red + down arrow + "−" sign for negative
- [ ] **Progress bar:** `role="progressbar"` with `aria-valuenow={percent}` during determinate phase; `aria-busy="true"` during indeterminate
- [ ] **Error table:** real `<table>` with `<th scope="col">` headers, `<td>` cells; screen readers announce structure
- [ ] **Button labels:** all buttons have visible text label (no icon-only buttons without `aria-label`)
- [ ] **Keyboard navigation:** Tab order is logical (drop zone → file input → buttons → error table if visible)
- [ ] **Skip links:** not required for single-page form (defer to layout-level concern)

### High Priority (UX polish)

- [ ] **Reduced motion:** Respect `prefers-reduced-motion: reduce` — disable spinner animation, progress bar transitions if requested
- [ ] **Error announcement:** Success/error cards use `aria-live="polite"` or `aria-live="assertive"` to notify screen readers
- [ ] **Loading state:** disable buttons during upload/parsing to prevent double-submission
- [ ] **Copy feedback:** "Copy all errors" button shows success state or brief toast (e.g., "Copied to clipboard!")

## Implementation Guidance

### DropZone Component

- Render a `<button>` with `role="button"` accepting click + drag-drop events
- Hidden file input: `<input type="file" accept=".csv,.txt" ref={inputRef} hidden />`
- `dragover` listener: add `class="drag-active"` to button (sets background)
- `drop` listener: prevent default, extract files, validate first file, trigger upload
- Validation errors render inline above the button (no full-page error)

### ProgressView Component

Two substates managed by parent `UploadPage`:
1. **Determinate (0–100%):** Display shadcn `Progress` with percent label
2. **Indeterminate (parsing):** Display Lucide `Loader2` spinning + "Parsing…" text

Switch happens when `xhr.upload.onload` fires; no API polling needed (single HTTP request contains final result).

### SuccessSummary Component

Receives: `{ filename, rowsInserted, durationMs, kpiDelta }`

Render:
- Green card with left border
- Green check icon + "Import successful" heading
- Meta line: `"{filename} · {rowsInserted} rows imported · completed in {durationMs / 1000}s"`
- KPI delta grid: 2×2 (desktop) or 1×4 (mobile) with before/after/delta columns
- Two buttons: "Go to Dashboard" (primary) + "Upload another file" (secondary)

**Styling:** Each KPI row shows:
- **Field name** (bold)
- **Before value** (if not null) or "—" (first import case)
- **After value** (always present)
- **Delta** with sign + arrow + color (green up / red down / neutral dash)

Example delta rendering:
```
Total Inventory Value
Before:  €0
After:   €125,000
Change:  +€125,000 ↑  (green text)
```

### ErrorSummary Component

Receives: `{ filename, errors[], durationMs }`

Render:
- Red card with left border
- Red alert icon + "Import failed" heading
- Grouped error summary: `"{errorCount} errors across {fieldCount} fields"`
- Bulleted list grouped by field (sorted by row count descending)
- Scrollable table with Row | Field | Value | Reason columns
- Two action buttons: "Copy all errors" + "Try another file"

**Copy function:** Use `navigator.clipboard.writeText()` to copy table as TSV (tab-separated values) for pasting into IT tickets or spreadsheets.

### Admin-Only Gate (Client-Side UX)

`ProtectedRoute` wraps the upload page. Inside the page component:
```ts
if (user?.role !== 'admin') {
  return <AdminAccessDenied />;
}
```

`AdminAccessDenied` renders: Card with heading "Admin access required" + explanatory text + no drop zone.

## Server-Side API Contract

**Endpoint:** `POST /api/v1/upload`

**Request:**
- Content-Type: `multipart/form-data`
- Field: `file` (single file, max 10 MB)

**Response (Success):** HTTP 200
```json
{
  "status": "success",
  "filename": "LagBes-2026-04-09.txt",
  "rowsInserted": 8925,
  "durationMs": 2340,
  "kpiDelta": {
    "totalInventoryValue": { "before": 0, "after": 3456789, "delta": 3456789 },
    "daysOnHand": { "before": null, "after": 45.2, "delta": 45.2 },
    "stockoutsCount": { "before": 0, "after": 12, "delta": 12 },
    "deadStockPct": { "before": null, "after": 8.5, "delta": 8.5 }
  }
}
```

**Response (Validation Error):** HTTP 400
```json
{
  "status": "failed",
  "filename": "LagBes-bad.txt",
  "rowsInserted": 0,
  "errors": [
    { "row": 42, "field": "Wert mit Abw.", "value": "1.234,56€", "reason": "Expected numeric value" },
    { "row": 99, "field": "letzt.Zugang", "value": "13.13.2025", "reason": "Invalid date format DD.MM.YY" }
  ],
  "durationMs": 150
}
```

**Response (Concurrent Upload):** HTTP 409 Conflict
```json
{ "error": "ingest_already_running", "message": "An ingest is already running — please wait a moment and try again." }
```

**Response (Auth Required):** HTTP 403 Forbidden
```json
{ "error": "admin_required", "message": "Admin role required to upload" }
```
